(1) git pull origin master (다른 pc나 github 웹에서 작업하여 github 저장소 내용이 바뀌었을 경우)
참조:
git fetch: 원격 저장소의 변경 사항을 가져오기만 하고, 자동으로 병합하지는 않습니다. 
git pull: 원격 저장소에서 변경 사항을 가져오면서 동시에 현재 로컬 브랜치와 자동으로 병합합니다. 

예시:
명령 (venv) PS C:\...\ANONYMOUS> git pull origin master

결과 From https://github.com/webstoryteller/anonymous_aws_django
     * branch            master     -> FETCH_HEAD
    Already up to date.

(3) git add .
(4) git commit -m "메시지"
(5) git push -u origin master


# 아래는 참조 사항 (git 사용이 생각보다 어려움)
... 로컬 PC에서 GitHub 원격 리포지토리로 복사 방법

git push --force 또는 git push --force-with-lease 사용하기 (주의 필요!)

이 방법은 로컬 리포지토리의 커밋 기록을 GitHub 원격 리포지토리의 커밋 기록과 완전히 덮어씁니다. 데이터 손실의 위험이 있으므로 협업 중인 팀이 있다면 사용하지 않아야 합니다. 

1. 로컬 리포지토리에서 작업 완료: 강제로 푸시할 로컬 리포지토리에서 모든 변경사항을 커밋합니다.

2. 원격 저장소 연결 확인: 로컬 리포지토리가 GitHub 원격 저장소에 제대로 연결되어 있는지 확인합니다. 연결되어 있지 않다면 git remote add origin [원격 저장소 주소] 명령어로 연결합니다.

3. 강제로 푸시: 터미널에서 git push --force 또는 git push --force-with-lease 명령어를 실행합니다.
git push --force: 로컬 브랜치의 내용을 원격 브랜치에 덮어씁니다.
git push --force-with-lease: 원격 브랜치가 변경되지 않은 경우에만 푸시하여, 다른 팀원이 푸시한 내용이 덮어씌워지는 것을 방지합니다.

4. 필요시 브랜치 강제 복사: 만약 로컬 브랜치와 원격 브랜치가 서로 다른 경우, git branch -f main (또는 다른 브랜치 이름) 명령으로 로컬 브랜치를 강제 복사한 후 푸시를 시도합니다.
